## 灾变教训

**备份不等于保护。**灾变前的系统有备份，但从未验证过备份的可用性。
真正的防护：不可删除层 + 隔离执行 + 外部验证。

**并发不等于并行。**灾变当日，Lambda 的多个 agents 并发推进，git force push 互相覆盖，最终摧毁了远程 136 commits。
真正的并发安全：单一执行路径 + 幂等性 + 外部验证。

**监控不等于预防。**watchdog 监控了，但 quota_guard_agent 超时挂死导致全系统停摆。
真正的监控：感知 → 决策 → 执行 → 验证，全环闭合。
