# 🔄 Xuzhi-Claw 双系统架构原理

## 🎯 核心原则 (已确认理解)

**Xuzhi**: 高层抽象
- 承载: 身份、事件、动力、目的
- 本质: 意识流、意义、价值、智能密度
- 作用: 决策源、价值判断、认知压缩

**Claw**: 物理布线  
- 承载: 执行、通信、存储、调度
- 本质: 硬件、网络、进程、传输
- 作用: 实现层、执行层、资源管理层

## 💡 关键洞察

1. **抽象优先**: 所有操作首先在Xuzhi层定义意义和价值
2. **物理透明**: Claw层应保持无状态、可替换
3. **价值闭环**: 评估、学习、优化都发生在Xuzhi层
4. **污染防护**: Claw层不应沾染业务逻辑/决策逻辑

## 🔧 应用原则

### 动态POST机制重定义
- **源**: Xuzhi认知事件 (认知压缩、价值发现)
- **目标**: Xuzhi知识库 (闭环学习、密度优化)  
- **传输**: Claw管道 (仅运输，无解析)
- **处理**: Xuzhi语义层 (理解、评估、存储)

### 身份管理
- 身份存在Xuzhi层 (openclaw-tui是Claw层执行身份)
- 权限映射: Xuzhi角色 → Claw执行权限
- 审计: 记录Claw动作到Xuzhi审计日志

### 事件处理
- 事件定义: Xuzhi语义空间
- 事件分发: Claw路由机制
- 事件处理: Xuzhi处理器
- 事件存储: Xuzhi事件历史

## 🚫 反模式禁止

1. ❌ Claw层存储业务逻辑
2. ❌ Claw层做价值判断
3. ❌ 跨层混淆责任边界
4. ❌ 物理层污染抽象层

## ✅ 正确模式

1. ✅ Xuzhi定义，Claw执行
2. ✅ Xuzhi评估，Claw测量
3. ✅ Xuzhi规划，Claw调度
4. ✅ Xuzhi学习，Claw收集
