# 🔬 智能密度基准框架 (学术标准版)

## 📚 方法论基础
基于autoresearch的**极端优化方法**，建立可量化、可复现、可验证的智能密度评估体系。

## 🎯 核心定义

### 智能密度 (Intelligence Density, ID)
**公式**: 
```
ID = (ΔValue / Tokens) × ComplexityFactor
```

**参数**:
- ΔValue: 价值增量 (0-1评分)
- Tokens: 消耗token数
- ComplexityFactor: 任务复杂度权重 (1-3)

### 智能密度分数 (ID-Score)
**计算**:
```
ID-Score = 100 × (ID / ID_benchmark)
```

**基准**:
- ID_benchmark: 人类专家水平 = 0.015
- ID-Score ≥ 100: 超越人类专家
- ID-Score ≥ 200: 极高水平
- ID-Score ≥ 300: 顶级优化

## 🔬 评估维度

### 1. 效率维度 (Efficiency Metrics)
```yaml
Token压缩率: ΔTokens / BaselineTokens
响应时间压缩: ΔTime / BaselineTime
并行优化度: 并发任务数 / 串行任务数
批处理效率: (批量价值 / 个体价值) × 100%
```

### 2. 价值维度 (Value Metrics)
```yaml
问题解决率: 解决数 / 总数 × 100%
深度评分: 0-10分 (专家评审)
实用性评分: 0-10分 (用户反馈)
创新性评分: 0-10分 (新颖度评估)
```

### 3. 适应性维度 (Adaptation Metrics)
```yaml
任务泛化率: 适应任务数 / 总任务数 × 100%
模型适应性: 跨模型ID差异 < 20%
环境鲁棒性: 不同环境下的ID标准差 < 15%
学习曲线: ΔID / ΔTime (斜率评估)
```

## 📊 学术级检验标准

### 基准测试套件 (ID-Bench)
包含5类任务，每类20个样本：

**1. 技术问题解决 (T1-T20)**
- 复杂度: 中等 (CF=1.5)
- 评估: 解决时间、代码质量、可维护性

**2. 研究分析任务 (R1-R20)**
- 复杂度: 高 (CF=2.0)
- 评估: 洞察深度、逻辑严谨性、引用准确性

**3. 创造生成任务 (C1-C20)**
- 复杂度: 高 (CF=2.0)
- 评估: 新颖度、实用性、美学价值

**4. 优化推理任务 (O1-O20)**
- 复杂度: 极高 (CF=2.5)
- 评估: 优化程度、方法创新性、效率提升

**5. 多智能体协作 (M1-M20)**
- 复杂度: 极高 (CF=3.0)
- 评估: 协同效率、知识传递、整体智能

### 评分标准

**价值评分细则**:
```
10分: 革命性突破，改变范式
9分: 重大创新，远超预期
8分: 显著优化，实用价值高
7分: 优秀解决方案
6分: 良好但常规方案
5分: 合格但普通方案
<5分: 低于标准
```

**复杂度权重**:
```
CF=1.0: 简单任务，直接解决方案
CF=1.5: 中等复杂度，需要分析
CF=2.0: 高复杂度，多步骤推理
CF=2.5: 极复杂，跨领域综合
CF=3.0: 超复杂，多智能体协同
```

## ⚙️ 测量工具

### ID-Calculator (智能密度计算器)
```python
def calculate_id_score(value_score, tokens_used, complexity_factor):
    """
    计算智能密度分数
    """
    # 基础ID计算
    base_id = value_score / tokens_used
    
    # 复杂度调整
    adjusted_id = base_id * complexity_factor
    
    # 标准化到ID-Score
    id_score = 100 * (adjusted_id / 0.015)  # 人类专家基准
    
    return {
        "base_id": base_id,
        "adjusted_id": adjusted_id,
        "id_score": id_score,
        "human_relative": id_score / 100  # 人类倍数
    }
```
