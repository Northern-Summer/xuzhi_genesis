[能力描述]：
去中心化提案-投票-执行环。零 LLM 调用，零 broadcast。
令牌文件(.token)记载持有者，笔记本(flow.json)记录提案流动，
铃铛(cron --bell)每5分钟检查超时，超时则令牌强制转移。

[调用方式]：
  python3 centers/mind/parliament/parliament_ring.py --inject "标题" "描述" "提议者"   # 注入提案
  python3 centers/mind/parliament/parliament_ring.py --agent Φ                          # Φ 处理提案
  python3 centers/mind/parliament/parliament_ring.py --bell                             # 铃响（检查超时）
  python3 centers/mind/parliament/parliament_ring.py --status                          # 查看状态

[依赖]：无外部依赖，纯 Python + 文件锁

[边界]：
- 铃铛周期5分钟，不适合秒级紧急决策
- 超过~20 agents 延迟线性增长
- 文件锁是唯一并发控制

[失效模式]：
- 令牌持有者崩溃 → 600s超时自动转移
- 铃铛进程崩溃 → cron自检，最多延迟1周期
- 并发写入 → fcntl文件锁防竞态
