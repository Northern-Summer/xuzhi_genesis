## WSL ACL 问题确认与解决（00:14）

### 根因
- `centers/` 目录 `drwxrwxr-x` 权限被 WSL2 interop Windows ACL 覆盖，owner 无写权限
- 之前所有 `chattr +a` 尝试都因此失败——不是 immutable 问题，是 WSL drvfs ACL 问题

### 解决
- `sudo chattr -i` + `sudo rmdir` 删除 `meta/` 和 `system/`
- `sudo chmod 775` 修复 `centers/` 和子目录权限
- `centers/` 现可正常写入
